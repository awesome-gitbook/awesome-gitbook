## API & Integration Platforms
 
- **[Gravitee Docs](https://docs.gravitee.io/)** — Documentation for the open-source API management platform. A GitBook flagship case study — Gravitee uses GitBook to scale docs across multiple teams while driving leads through search. Covers API gateway configuration, access management, and developer portal setup with a polished, professional structure.
 
- **[Bird API Docs](https://docs.bird.com/api)** — API documentation for the communications platform (formerly MessageBird). Clean API reference covering SMS, voice, email, and chat channels. A strong example of using GitBook for multi-product API documentation with consistent formatting across endpoints.
 
- **[bunq API Docs](https://doc.bunq.com/)** — Developer documentation for the European neobank's API. Notable for its clear authentication guides, sandbox environment docs, and well-organized endpoint references. A great example of fintech API docs done right on GitBook.
 
- **[Cortex Docs](https://docs.cortex.io/)** — Documentation for the internal developer portal platform. Stands out by combining product docs, API references, and help center content in a single GitBook site. Clean navigation across a broad feature set including scorecards, service catalogs, and integrations.
 
## Cloud & Infrastructure
 
- **[IONOS Cloud Docs](https://docs.ionos.com/cloud)** — Documentation for one of Europe's largest cloud and hosting providers. Covers compute, storage, networking, Kubernetes, and managed databases. A good example of enterprise-scale product documentation on GitBook, with clear separation between product guides and API references.
 
- **[Erigon Docs](https://erigon.gitbook.io/erigon)** — Documentation for the Ethereum execution client. Covers installation, advanced usage, consensus layer configuration, and performance tuning. Practical, developer-focused infrastructure docs.
 
## Data Governance & Analytics
 
- **[Immuta Documentation](https://documentation.immuta.com/)** — Comprehensive docs for the data security and governance platform. A GitBook case study — Immuta migrated to GitBook for better cross-team collaboration and saw immediate results. Covers data access control, policy authoring, and platform integrations across Snowflake, Databricks, and more. Rich with governance frameworks and strategic implementation guides.
 
- **[Exploratory Data Analysis Guide](https://drshahizan.gitbook.io/eda1)** — An academic GitBook covering EDA techniques with real-world case studies spanning healthcare, finance, environmental data, and e-commerce. Demonstrates how GitBook works well for structured educational content with practical examples.
 
## Developer Tools & Security
 
- **[Snyk User Docs](https://docs.snyk.io/)** — One of the highest-profile GitBook deployments. Snyk's documentation is a masterclass in organizing a complex security platform into digestible sections. Covers CLI, IDE integrations, container scanning, and more — all with a consistent voice and clear navigation. Snyk has publicly shared their GitBook workflow as a case study.
 
- **[Blockscout Docs](https://docs.blockscout.com/)** — Documentation for the open-source blockchain explorer. Well-structured with clear deployment guides, API references, and customization instructions. A good example of documenting a self-hosted open-source project.
 
## Automation & Low-Code
 
- **[Make Developer Docs](https://developers.make.com/)** — Developer documentation for the automation platform (formerly Integromat). Covers custom app development, API integration, and module building. A solid example of using GitBook to document a platform that bridges technical and non-technical audiences.

- **[Rierino Docs](https://docs.rierino.com/)** — Extensive documentation for the low-code microservices development platform. Stands out for its depth across both DevOps and business logic topics, with clear diagrams and parameter reference tables throughout.
 
- **[Relay.app Docs](https://docs.relay.app/)** — Documentation for the collaborative workflow automation tool. GitBook customer story highlight — the team praised GitBook for its low barrier to contribution, allowing anyone on the team to add content while maintaining a consistent look and structure.
 
## Fintech & Payments
 
- **[Xplor Pay Docs](https://docs.xplorpay.com/)** — Documentation for the embedded payments platform. A GitBook case study — Xplor Pay credits their documentation as a competitive advantage. Covers payment integration guides, API references, and merchant onboarding flows.
 
- **[Keeper Docs](https://newdocs.keeper.io/)** — Documentation for the password management and cybersecurity platform. Covers deployment guides, admin console configuration, and enterprise SSO setup. A good example of security-focused product documentation with clear audience segmentation.
 
## Blockchain & Web3
 
- **[Aave Developer Docs](https://docs.aave.com/)** — Comprehensive developer documentation for one of the largest DeFi lending protocols. Covers V1 through V3 with clearly versioned docs, smart contract references, integration guides, and risk parameters. A benchmark for how to document a protocol that evolves across major versions.
 
- **[Hyperliquid Docs](https://hyperliquid.gitbook.io/hyperliquid-docs)** — Clean and focused documentation for the high-performance L1 blockchain. Explains the HyperBFT consensus algorithm, HyperCore and HyperEVM architecture, and API details. Excellent example of concise but thorough protocol documentation.
 
- **[PancakeSwap Docs](https://docs.pancakeswap.finance/)** — Product documentation for one of the largest DEXs. Notable for balancing end-user guides with developer/protocol documentation in a single site, covering trading, farming, staking, and lottery features with user-friendly language.
 
- **[Chainlink CCIP Masterclass](https://andrej-rakic.gitbook.io/chainlink-ccip)** — A standout educational GitBook that walks developers through Chainlink's Cross-Chain Interoperability Protocol step by step. Features hands-on exercises, architecture diagrams, and code samples for Hardhat, Foundry, and Remix. Great example of a tutorial-style GitBook.
 
- **[Lido Docs](https://docs.lido.fi/)** — Documentation for the liquid staking protocol. Stands out for its exhaustive deployed contracts reference across multiple chains, governance documentation, and integration guides.
 
## Game Development
 
- **[Godot Essentials Addons](https://godot-essentials.gitbook.io/addons-documentation)** — Documentation for a Godot game engine plugin suite. Covers components like health systems, state machines, camera effects, and movement controllers. Notable for using emoji-based navigation and providing runnable examples, including a full Celeste-style platformer demo.
 
- **[Sway Charts Pro Scripts](https://sway-technologies.gitbook.io/sway-charts-pro-scripts-documentation)** — API documentation for a trading chart platform's scripting framework. Covers widget development, script lifecycle, and Promise-based APIs. A niche but well-executed example of documenting a domain-specific scripting environment.
 
## Learning Resources
 
- **[How to Write Great Documentation](https://steves-space.gitbook.io/how-to-write-great-documentation)** — A meta-resource: a GitBook about writing documentation, with tips on structure, editing, and maintenance. Useful both for its content and as a template for how to organize a guide-style GitBook.
 
- **[7 Code Documentation Best Practices](https://agile-7.gitbook.io/7-code-documentation-best-practices)** — A concise guide on writing effective code documentation. Covers audience analysis, examples, structure, and maintenance. Useful as a quick-reference companion for any developer.
 
- **[Blockchain Development Guide (devpill.me)](https://github.com/dcbuild3r/blockchain-development-guide)** — A public good learning resource hosted on GitBook covering the full developer journey — from cryptography fundamentals to smart contract development. Rich in curated links and structured learning paths.
 
## GitBook's Own Docs
 
- **[GitBook Documentation](https://docs.gitbook.com/)** — GitBook's own documentation is, naturally, a showcase of the platform's capabilities. Features AI-powered search, structured guides, API references, and best practices. The gold standard reference for what GitBook can do.
 
- **[GitBook Guides](https://docs.gitbook.com/guides)** — A collection of how-to guides covering API documentation, analytics, personalization, branding, and open-source contributions. Worth studying for structure and writing style.

## What Makes a GitBook "Awesome"?
 
When curating this list, we looked for documentation that excels in one or more of the following:
 
- **Clear information architecture** — Logical grouping of pages, good use of sections and sub-pages
- **Rich content** — Code samples, diagrams, tables, embedded media, and interactive elements
- **Consistent voice** — Professional, readable writing that stays consistent across the site
- **Versioning & maintenance** — Docs that keep up with product changes and version their content
- **Developer experience** — Quick-start guides, copy-friendly code blocks, and API references
- **Accessibility** — Works well for both beginners and advanced users
